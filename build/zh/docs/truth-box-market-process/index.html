运行流程 (Process Flow)

系统是如何运作的？​

我们将拆解从“证据铸造”到“买家博弈”再到“最终公开”的完整生命周期。

在协议运行中，涉及三类核心角色：吹哨人 (Minter)、Buyer 以及Helpers: Seller/Completer。

1. 真相资产化 (Assetization)​

  • 上传：上传证据文件与元数据至 IPFS/Arweave。
  • 铸造：在智能合约中铸造 Truth Box。

2. 市场流通 (Market Circulation)​

  • 出售/拍卖：Minter 可在保护期内发起出售或拍卖。
  • Help Sell：若超时，任何人都可以辅助出售，成为Seller。

3. 交割与验证 (Delivery)​

  • 购买：Buyer 支付后，获得访问权。
  • 验货期：Buyer 可选择申请退款，或者完成订单；
  • Help Complete：若超时，任何人都可以辅助完成订单，成为Completer。

4. 延迟披露与博弈 (Delayed Disclosure & Game)​

  • 延迟状态：交易完成后，Box进入“Delaying”状态。
  • 延迟费用：Buyer 需定期支付延迟费用 (Delayed Fee)。
  • 最终披露：如果 Buyer 停止支付延迟费用，Box将自动进入“Published”状态，真相被披露，完成其最终的社会价值兑现。

实操 SOP（用户快速上手）​

以下流程用于帮助用户在不同网络下，以尽可能高的隐私级别完成操作。

A. 主网 SOP（隐私优先，推荐）​

  1. 准备钱包并妥善保存私钥/助记词；如你是新用户，可使用平台自动创建的钱包（仅用于签名）。
  2. 连接主网后，开启代理交互（EIP-712 中继）。
  3. 上传证据前，先在本地完成加密与文件粉碎，再上传到 IPFS/Arweave。
  4. 使用签名请求完成铸造、上架、购买、延长等操作，避免钱包地址直接发起链上交易。
  5. 定期检查 Box 状态与截止时间（尤其是退款期、延迟披露期限），按需执行后续动作。

主网开启代理交互后，钱包地址不会留下直接的链上交互记录，更适合对隐私有高要求的真实场景。

B. 测试网 SOP（调试优先）​

  1. 使用常规 Web3 钱包连接测试网并直接交互合约。
  2. 按照创建、交易、退款、延迟披露等完整流程进行功能验证。
  3. 测试网默认不启用主网代理交互机制，方便 Web3 用户调试与排查问题。

C. 无钱包用户 SOP（面向非 Web3 用户）​

  1. 首次进入时，系统会自动创建一个用于签名的空钱包。
  2. 该钱包密钥会保存在浏览器 localStorage 中，同时你仍需自行做好密钥备份与安全管理。
  3. 得益于 Oasis Sapphire 的低 Gas 费，每个钱包地址默认提供一次免费铸造机会，便于首次体验。

请务必理解：自动创建钱包是“降低门槛”方案，不等于“托管安全”。一旦浏览器数据丢失且无备份，可能无法恢复访问权限。
